ESTRUCTURA

PASOS DE UN DISEÑO

El diseño de un sistema es el modelo general de ese sistema, contiene todas las especificaciones que dan al sistema su forma y estructura dichas especificaciones deben contemplar todos los componentes de administración, organización y tecnología del sistema a implementar.

DESARROLLO DEL SISTEMA Y DOCUMENTACION

Consiste en la traducción del diseño a una forma entendible para la máquina, lo que se logra a través de código fuente usando el talento de los programadores que lo elaboran para que satisfaga las especificaciones del diseño.

La documentación consiste en plasmar en un documento escrito todas las especificaciones técnicas del diseño con el objetivo que otros analistas y programadores entiendan como se diseñó y desarrollo y sea fácil de entender para poder mantenerlo y mejorarlo.

PRUEBAS DEL SISTEMA

Una vez desarrollado el sistema es necesario realizar pruebas primero parte por parte y después en un todo, seleccionando a un usuario habitual que navegue por todos los módulos aun en incluyendo información que no se espera que un usuario en condiciones normales ingrese, todo esto para saber cómo se comporta cada parte del sistema aún en condiciones no previstas, todo esto con el fin de hacer las correcciones inmediatamente antes de finalizar esta fase y no cuando el sistema entra en operación porque en este caso las correcciones son costosas y generan desconfianza en el usuario final.

IMPLEMENTACION

Consiste en ubicar la aplicación en las maquinas previstas efectuando la instalación, luego se prepara la información inicial que se incluirá para que arranque el sistema. Posteriormente se procede con la capacitación de los usuarios de modo que las dudas sean mínimas. En esta fase el clave llevar u proceso de empalme con el sistema anterior y debe quedar claro que el nuevo sistema por si solo desplaza al anterior.
